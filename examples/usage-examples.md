## Example 5: Thinking, Fast and Slow (Low Fit Example)

**User Request:**
> "Turn Kahneman's 'Thinking, Fast and Slow' into skills"

**Book2Skill Factory Analysis:**

### Fit Assessment

```
Fit Level: ❌ LOW

Reasoning:
- Academic research, not a framework book
- Descriptive (how people think) vs prescriptive (what to do)
- Highly abstract, cognitive biases difficult to operationalize
- Better as reference/conceptual understanding

Recommended Approach: Create decision-making heuristics instead
```

### Alternative: Extract Actionable Parts

Instead of full book conversion, extract specific tools:

1. `bias-detection` - Identify cognitive biases in decisions
2. `system-2-thinking` - Activate slow thinking for important decisions
3. `prospect-theory` - Apply to pricing/risk decisions

**Challenge:** Limited actionability
**Recommendation:** Use for awareness, not as primary skill system

## Testing Your Skills

**After conversion, test with:**

1. **Trigger Test**
   ```
   "Use [skill-name] to [problem]"
   Example: "Use ERRC framework to escape price wars"
   Expected: Skill triggers correctly
   ```

2. **Independent Test**
   ```
   Can skill be used without reading the book?
   Example: Apply to your own industry without book knowledge
   Expected: Yes, workflow is self-contained
   ```

3. **Value Test**
   ```
   Does skill produce actionable output?
   Example: Run skill, get specific decisions/recommendations
   Expected: Yes, concrete next steps
   ```

## Common Pitfalls

❌ **Pitfall 1:** Creating one giant skill for the whole book
   ✅ **Fix:** One book = One plugin package with 4-12 skills

❌ **Pitfall 2:** Writing book summaries instead of skills
   ✅ **Fix:** Focus on workflows and actionable tools

❌ **Pitfall 3:** Skipping fit assessment
   ✅ **Fix:** Always assess book suitability first

❌ **Pitfall 4:** Including every chapter
   ✅ **Fix:** Only extract frameworks/tools, not narrative content

❌ **Pitfall 5:** No real-world examples
   ✅ **Fix:** Include 2-3 case studies per skill

## Success Metrics

A successful book2skill conversion should achieve:

✅ **Triggerable** - Each skill activates on relevant user requests
✅ **Independent** - No book reading required to use skills
✅ **Actionable** - Produces specific decisions/recommendations
✅ **Validated** - Tested on real use cases
✅ **Complete** - 40,000-80,000 words, 4-12 skills, 2-5 templates

---

**Remember:** The goal is not to summarize books, but to create **executable skill systems** that help users apply methodologies in real-world scenarios. 🚀
