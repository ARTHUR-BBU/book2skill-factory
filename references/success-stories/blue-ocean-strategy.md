**Problem:** Products stuck in "better-cheaper" competition, price wars, commoditization

**Skill Body Structure:**

```markdown
# Eliminate-Reduce-Raise-Create (ERRC Framework)

Redesign your value curve to escape competition by asking: What should we eliminate? Reduce? Raise? Create?

## Quick Start (3 minutes)

1. List all competitive factors in your industry
2. For each factor, ask the 4 ERRC questions
3. Make decisions and create new value curve
4. Validate: Does it achieve differentiation AND low cost?

## When to Use This Skill

**Trigger scenarios:**
- "How do I escape price wars?"
- "My product looks like competitors' - how to differentiate?"
- "Stuck in 'better-cheaper' trap"
- "Need breakthrough value curve"

## Step-by-Step Workflow

### Step 1: List Competitive Factors (5 minutes)

**What to do:** Brainstorm all factors that your industry competes on.

**Examples:**
- Price, quality, features, service, speed, design, brand...

**Guided questions:**
- What do competitors advertise?
- What do customers compare?
- What factors appear in industry reviews?

### Step 2: Apply ERRC Questions (15 minutes)

**For EACH factor, ask 4 questions:**

**Eliminate:** Which factors should be eliminated because they no longer create value?
- Example: Yellow Tail eliminated enological terminology and complexity

**Reduce:** Which factors should be reduced below industry standard?
- Example: Yellow Tail reduced wine selection to 2 varieties

**Raise:** Which factors should be raised above industry standard?
- Example: Yellow Tail raised sweetness and fruitiness

**Create:** Which factors should be created that never existed before?
- Example: Yellow Tail created fun, easy-drinking positioning

### Step 3: Make Decisions (10 minutes)

**Fill out ERRC matrix:**

| Factor | Eliminate? | Reduce? | Raise? | Create? |
|--------|-----------|---------|--------|---------|
| Price  |           | ✓ Below |        |         |
| Quality|           |         | ✓ Above|         |
| ...    |           |         |        |         |

### Step 4: Validate (5 minutes)

**Check: Does your new value curve achieve:**
- ✅ Differentiation from competitors?
- ✅ Low cost structure (via eliminate/reduce)?

If NO → Revisit ERRC decisions.

## Common Mistakes to Avoid

❌ **Mistake 1:** Skipping eliminate/reduce
   ✅ **Fix:** Start with eliminate/reduce - that's where most value innovation comes from

❌ **Mistake 2:** Only raising/creating (adds cost)
   ✅ **Fix:** Must eliminate/reduce to fund raise/create

❌ **Mistake 3:** Copying competitors
   ✅ **Fix:** ERRC is about divergence, not convergence

## Real-World Example: Yellow Tail Wine

**Before:** Commodity wine market, 8,000+ varieties, complex terminology, premium pricing

**ERRC Analysis:**
- **Eliminate:** Enological terminology, aging qualities, vineyard prestige
- **Reduce:** Wine complexity, selection range (down to 2 varieties)
- **Raise:** Sweetness, fruitiness, approachability
- **Create:** Fun, easy-drinking positioning for beer drinkers

**Result:** $1B+ brand, fastest-growing wine in history

## Template

See `assets/templates/errc-matrix-template.md` for ready-to-use worksheet.

## See Also

- `references/errc-framework.md` - Complete 8,000-word guide
- `strategy-canvas` - Visualize your value curve vs competitors
- `value-innovation-check` - Validate your ERRC decisions
```

## 🎓 Key Learnings from This Conversion

### What Worked Well

1. **Starting with fit assessment**
   - Saved time by confirming high fit upfront
   - Clear scope: 6 skills matching 6 frameworks

2. **Progressive disclosure architecture**
   - SKILL.md for quick adoption (10,000 words)
   - references/ for deep dives (36,000 words)
   - Templates for immediate action

3. **Rich skill descriptions**
   - 3-4 trigger scenarios per skill
   - Optimized keywords for Claude recognition
   - Tested successfully in practice

4. **Real-world validation**
   - Classic cases from book (Yellow Tail, Cirque du Soleil)
   - Modern application (AI short drama market)
   - Demonstrated tool effectiveness

5. **Structured templates**
   - Ready-to-use worksheets
   - Guided questions throughout
   - Examples for every section

### Challenges Faced

1. **Reference writing time**
   - 240 minutes for 6 references (40 min each)
   - Solution: Created template structure first, then filled in content

2. **Balancing depth vs accessibility**
   - Too deep = intimidating
   - Too shallow = not useful
   - Solution: Progressive disclosure (SKILL.md vs references/)

3. **Avoiding book summary trap**
   - Temptation to include everything from book
   - Solution: Only extract actionable frameworks/tools

4. **Cross-skill consistency**
   - Ensuring terminology aligns across 6 skills
   - Solution: Created glossary, standardized terminology upfront

### Improvements for Next Time

1. **Template creation first**
   - Create templates before writing references
   - Templates inform reference structure

2. **Skill testing earlier**
   - Test skill triggers after writing first 2 skills
   - Catch description issues early

3. **Example bank**
   - Collect 5-10 examples upfront
   - Distribute across all skills

4. **Version control**
   - Use git from start
   - Track iterations more systematically
